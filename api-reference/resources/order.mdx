---
title: "Order"
description: "Order resource schema"
---

<ResponseField name="id" type="string">
  Unique identifier for the order
</ResponseField>

<ResponseField name="quote_id" type="string">
  ID of the associated quote
</ResponseField>

<ResponseField name="offering_id" type="string">
  ID of the associated offering
</ResponseField>

<ResponseField name="reference_number" type="string">
  Internal reference number for the order
</ResponseField>

<ResponseField name="external_reference_id" type="string">
  External reference ID provided by the organization
</ResponseField>

<ResponseField name="exchange_rate" type="string">
  The exchange rate used for this order
</ResponseField>

<ResponseField name="effective_exchange_rate" type="string">
  The actual effective exchange rate after execution
</ResponseField>

<ResponseField name="payin" type="object">
  Payment input details
</ResponseField>

<ResponseField name="payin.external_payment_instrument" type="object">
  External payment instrument object for incoming payments
</ResponseField>

<ResponseField name="payin.total" type="string">
  Total amount paid in
</ResponseField>

<ResponseField name="payin.fee" type="string">
  Fee amount
</ResponseField>

<ResponseField name="payin.tax" type="string">
  Tax amount
</ResponseField>

<ResponseField name="payout" type="object">
  Payment output details
</ResponseField>

<ResponseField name="payout.external_payment_instrument" type="object">
  External payment instrument object for outgoing payments
</ResponseField>

<ResponseField name="payout.total" type="string">
  Total amount to be paid out
</ResponseField>

<ResponseField name="status" type="object">
  Current status information
</ResponseField>

<ResponseField name="status.current_status" type="string">
  Current status of the order (see Order Statuses)
</ResponseField>

<ResponseField name="status.status_history" type="object[]">
  Historical record of status changes
</ResponseField>

<ResponseField name="status.status_history[].status" type="string">
  The status value for this history entry
</ResponseField>

<ResponseField name="status.status_history[].created_at" type="string">
  Timestamp when this status change occurred
</ResponseField>

<ResponseField name="funding_instructions" type="object">
  Instructions for funding the order. Use this to show the end user where and how to send funds (e.g. bank details for NGN). Structure depends on `funding_instructions.currency` and `funding_instructions.method`.
</ResponseField>

<ResponseField name="funding_instructions.currency" type="string">
  Currency for funding (see Currency Symbols). Determines the shape of `details`.
</ResponseField>

<ResponseField name="funding_instructions.method" type="string">
  Payment method for funding (see Payment Methods). For NGN orders this is `NGN_BANK_ACCOUNT`.
</ResponseField>

<ResponseField name="funding_instructions.details" type="object">
  Payment method-specific funding details. Echoes the payin external payment instrument. For **NGN** (`currency` NGN, `method` NGN_BANK_ACCOUNT): `bank_name`, `bank_account_number`, `doc_id`, `account_holder_first_name`, `account_holder_last_name`, `account_holder_business_name`. For other currencies (e.g. MXN, BRL), the structure differs.
</ResponseField>

### Funding instructions by currency

| Currency | Method | Details shape |
| -------- | ------ | -------------- |
| NGN | `NGN_BANK_ACCOUNT` | `bank_name`, `bank_account_number`, `doc_id`, `account_holder_first_name`, `account_holder_last_name`, `account_holder_business_name` |
| MXN | Bank transfer | CLABE, reference, bank name, account holder |
| BRL | PIX | Pix copy-paste code, QR code (base64) |
| USDC | `SOLANA_ADDRESS` | Blockchain address and reference |

<ResponseField name="retry_order_id" type="string">
  ID of the retry order if this order was retried
</ResponseField>

<ResponseField name="created_at" type="string">
  Timestamp when the order was initially created
</ResponseField>

<ResponseField name="updated_at" type="string">
  When the order was last updated
</ResponseField>
