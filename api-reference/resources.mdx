# Resources

## Organization

An organization represents the business entity that has a direct engagement with Stable Sea to use our API or other services.

```json
{
  "id": "string",
  "name": "string"
}
```

## Hours of Operation

```json
{
  "liquidity_provider": "string",
  "payin_currency": "string",
  "payout_currency": "string",
  "hours_of_operation": [
    {
      "calendar_date": "string",
      "open_time": "string",
      "close_time": "string"
    }
  ]
}
```

## Customer

A customer represents the customer _of an organization_. A customer must be created by the respective organization prior to leveraging Stable Sea's APIs to facilitate payouts or other services on behalf of that customers.

The customer will undergo KYX procedures prior to being approved by Stable Sea.

<Warning>
  A customer that represents the organization itself will automatically be
  created when the organization is successfully onboarded. This customer can be
  used to submit RFQs on behalf of the organization itself.
</Warning>

```json
{
  "id": "string",
  "name": "string",
  "status": {
    "current_status": "string", // see Customer Statuses
    "status_history": [
      {
        "status": "string",
        "created_at": "string"
      }
    ]
  },
  "created_at": "string",
  "updated_at": "string"
}
```

## External Payment Instrument

An External Payment Instrument represents a payment instrument that a customer can use to receive payouts.

```json
{
  "id": "string",
  "alias": "string",
  "currency": "string", // see Currency Symbols
  "method": "string", // see Payment Methods
  "status": {
    "current_status": "string", // see External Payment Instrument Statuses
    "status_history": [
      {
        "status": "string",
        "created_at": "string"
      }
    ]
  },
  "details": "object", // the structure is unique to the given method, see ./payment-instrument-details
  "created_at": "string",
  "updated_at": "string"
}
```

## Offering

An offering represents a instructive material for placing an order.

```json
{
  "id": "string",
  "payin": {
    "external_payment_instrument_id": "string",
    "limits": {
      "min": "string",
      "max": "string"
    },
    "indicative_cost_tiers": [
      {
        "min": "string",
        "max": "string",
        "bps": "string"
      }
    ]
  },
  "payout": {
    "external_payment_instrument_id": "string"
  },
  "indicative_exchange_rate": "string",
  "current_day_hours_of_operation": {
    "open": "string",
    "close": "string"
  },
  "liquidity_provider": "string",
  "estimated_settlement_time": "string",
  "created_at": "string"
}
```

### Order Statuses

| Status                            | Description                                                                            |
| --------------------------------- | -------------------------------------------------------------------------------------- |
| `STEP_1_AWAITING_FUNDING`         | Order submitted but funds have not arrived                                             |
| `STEP_2_FILLING_FX`               | Funds have arrived in time and FX is being filled                                      |
| `STEP_3_SETTLING_PAYOUT`          | FX has been filled and final payout is in progress                                     |
| `STEP_4_COMPLETED`                | Terminal success state                                                                 |
| `FAILED_FUNDING_EXPIRED`          | Failure state where funds did not arrive by the Quote's `funding_expires_at` timestamp |
| `FAILED_DECLINED_BY_LIQ_PROVIDER` | Failure state where discretionary measure taken by the Liquidity Provider              |
| `FAILED_REFUND_REQUESTED`         | Failure state wherein a refund was requested by the Customer                           |
| `FAILED_REFUND_PROCESSED`         | Terminal failure state where the refund has settled                                    |
